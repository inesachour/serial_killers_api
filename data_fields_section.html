Understanding the data fields returned by the API and their formats.

Basic Information

id (integer) - Unique identifier for the serial killer

name (string) - Full legal name of the subject

alias (string, optional) - Known alias or nickname
Example: "The Night Stalker", "BTK Killer"

Personal Details

date_of_birth (date, optional) - Birth date in YYYY-MM-DD format

birth_country (string, optional) - Country of birth
Example: "USA", "United Kingdom"

gender (string, optional) - Gender of the subject
Values: "male", "female", "unknown"

Criminal Activity

proven_victims (integer, optional) - Confirmed number of victims

possible_victims (string, optional) - Estimated victim count
‚ö†Ô∏è Note: Can be a number ("17") or text with "+" ("100+") indicating minimum count

years_active (string, optional) - Period of criminal activity
‚ö†Ô∏è Format examples:
‚Ä¢ "1990-2001" = Active continuously from 1990 to 2001
‚Ä¢ "1990,2001" = Active in 1990 and 2001 (non-continuous)
‚Ä¢ "1980-1985,1990-1995" = Multiple active periods

active_countries (string, optional) - Countries where crimes occurred
Example: "USA, Canada" for multiple countries

modus_operandi (string, optional) - Method of operation
Example: "Strangulation", "Poisoning"

Legal Status

capture_date (date, optional) - Date of apprehension in YYYY-MM-DD format

sentence (string, optional) - Legal sentence received
Example: "Life imprisonment", "Death penalty"

status (string, optional) - Current status
Values: "Imprisoned", "Deceased", "At Large", "Released", "Unknown"

death_date (date, optional) - Date of death in YYYY-MM-DD format
Only present if status is "Deceased"

manner_of_death (string, optional) - How the subject died
Example: "Executed", "Natural causes", "Suicide"

Additional Information

notes (string, optional) - Additional details or context about the case

Field Interpretation Guidelines
